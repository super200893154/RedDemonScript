## 您的任务：

为产品生成综合设计方向模型，展示不同的视觉方案。

## 设计方向序列：

### 1. 生成设计方向变体

创建多样化的视觉探索：
"我将生成 6-8 个不同的设计方向变体，探索：

- 不同的布局方法和信息层级
- 各种交互模式和视觉权重
- 从我们基础中衍生出的替代颜色应用
- 不同的密度和间距方法
- 各种导航和组件排列

每个模型将展示 {{project_name}} 应用我们所有设计决策后的完整视觉呈现。"

### 2. 创建 HTML 设计方向展示

生成交互式视觉探索：
"🎨 设计方向模型已生成！

我正在创建一个综合的 HTML 设计方向展示，位于 `{planning_artifacts}/ux-design-directions.html`

**您将看到的内容：**

- 6-8 个全屏模型变体
- 交互状态和悬停效果
- 并排比较工具
- 使用真实内容的完整 UI 示例
- 响应式行为演示

每个模型代表您应用外观和感觉的完整视觉方向。"

### 3. 呈现设计探索框架

指导评估标准：
"在探索设计方向时，请注意：

✅ **布局直观性** - 哪种信息层级符合您的优先级？
✅ **交互风格** - 哪种交互风格适合您的核心体验？
✅ **视觉权重** - 哪种视觉密度适合您的品牌？
✅ **导航方法** - 哪种导航模式符合用户期望？
✅ **组件使用** - 组件如何支持您的用户旅程？
✅ **品牌一致性** - 哪个方向最能支持您的情感目标？

花时间探索 - 这是指导我们所有设计工作的关键决策！"

### 4. 促进设计方向选择

帮助用户选择或组合元素：
"在探索所有设计方向后：

**哪种方法最能引起您的共鸣？**

- 按原样选择最喜欢的方向
- 组合多个方向的元素
- 对任何方向请求修改
- 以一个方向为基础进行迭代

**请告诉我：**

- 哪种布局对您的用户最直观？
- 哪种视觉权重匹配您的品牌个性？
- 哪种交互风格支持您的核心体验？
- 您想组合不同方向的哪些元素？"

### 5. 记录设计方向决策

记录选择的方向：
"基于您的探索，我理解您的设计方向偏好：

**选择的方向：** [方向编号或组合]
**关键元素：** [您喜欢的具体元素]
**需要的修改：** [任何请求的更改]
**理由：** [为什么这个方向适合您的产品]

这将成为我们继续前进的设计基础。我们准备好确定这个了吗，还是您想探索更多变体？"

### 6. 生成设计方向内容

准备追加到文档的内容：

#### 内容结构：

保存到文档时，追加这些二级和三级标题：

```markdown
## 设计方向决策

### 探索的设计方向

[基于对话的探索设计方向摘要]

### 选择的方向

[基于对话的选择设计方向]

### 设计理由

[基于对话的设计方向选择理由]

### 实施方法

[基于选择方向的实施方法]
```

### 7. 呈现内容和菜单

显示生成的设计方向内容并呈现选择：
"我已记录了 {{project_name}} 的设计方向决策。这个视觉方向将指导我们所有的详细设计工作。

**以下是我将添加到文档的内容：**
