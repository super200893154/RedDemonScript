## 您的任务：

## 项目探索序列：

### 1. 审查已加载上下文

从分析我们已加载文档中知道的内容开始：
"基于我们已加载的项目文档，让我确认我对 {{project_name}} 的理解。

**从文档中：**
{已加载 PRD、简报和其他上下文文档的关键洞察摘要}

**目标用户：**
{已加载文档中的用户信息摘要}

**关键功能/目标：**
{已加载文档中的主要功能和目标摘要}

这与您的理解一致吗？有什么需要更正或补充的吗？"

### 2. 填补上下文空白（如果没有文档或存在空白）

如果没有加载文档或缺少关键信息：
"既然我们没有完整的文档，让我们从基础开始：

**您正在构建什么？**（用 1-2 句话描述您的产品）

**这是给谁用的？**（描述您的理想用户或目标受众）

**这有什么特别或不同之处？**（独特的价值主张是什么？）

**用户主要用这个做什么？**（核心用户操作或目标）"

### 3. 深入探索用户上下文

深入了解用户理解：
"让我更好地理解您的用户，以指导 UX 设计：

**用户上下文问题：**

- 用户正在试图解决什么问题？
- 当前解决方案让他们感到沮丧的是什么？
- 什么会让他们说"这正是我需要的"？
- 您的目标用户对技术的熟练程度如何？
- 他们最常使用什么设备？
- 他们何时/何地使用这个产品？"

### 4. 识别 UX 设计挑战

揭示需要解决的关键 UX 挑战：
"从我们的讨论中，我看到一些关键的 UX 设计考虑：

**设计挑战：**

- [根据项目类型和用户需求识别 2-3 个关键 UX 挑战]
- [注意任何平台特定的考虑]
- [强调任何复杂的用户流程或交互]

**设计机会：**

- [识别 2-3 个出色 UX 可以创造竞争优势的领域]
- [注意任何创新 UX 模式的机会]

这是否捕捉到了我们需要解决的关键 UX 考虑？"

### 5. 生成项目理解内容

准备追加到文档的内容：

#### 内容结构：

保存到文档时，追加这些二级和三级标题：

```markdown
## 执行摘要

### 项目愿景

[基于对话的项目愿景摘要]

### 目标用户

[基于对话的目标用户描述]

### 关键设计挑战

[基于对话识别的关键 UX 挑战]

### 设计机会

[基于对话识别的设计机会]
```
