## 灵感分析序列：

### 1. 识别用户喜爱的应用

从收集灵感来源开始：
"让我们从目标用户已经喜爱和频繁使用的产品中学习。

**灵感问题：**

- 说出 2-3 个您的目标用户已经喜爱并经常使用的应用
- 对于每个应用，从 UX 角度它们做得好在哪里？
- 什么让体验引人注目或令人愉悦？
- 什么让用户不断回到这些应用？

考虑您所在类别的应用，甚至是有出色 UX 的不相关产品。"

### 2. 分析 UX 模式和原则

分解这些应用成功的原因：
"对于每个启发性应用，让我们分析它们的 UX 成功：

**对于 [应用名称]：**

- 它优雅地解决了什么核心问题？
- 它的入门体验为什么有效？
- 它如何处理导航和信息层级？
- 它最创新或最令人愉悦的交互是什么？
- 什么视觉设计选择支持用户体验？
- 它如何处理错误或边缘情况？"

### 3. 提取可迁移模式

识别可以应用于您项目的模式：
"**可迁移 UX 模式：**
纵观这些启发性应用，我看到一些我们可以适配的模式：

**导航模式：**

- [模式 1] - 可能适合您的 [特定用例]
- [模式 2] - 可能解决您的 [特定挑战]

**交互模式：**

- [模式 1] - 非常适合 [您的用户目标]
- [模式 2] - 解决 [您的用户痛点]

**视觉模式：**

- [模式 1] - 支持您的 [情感目标]
- [模式 2] - 与您的 [平台需求] 一致

这些模式中哪些对您的产品最有共鸣？"

### 4. 识别要避免的反模式

根据分析提出不应做的事情：
"**要避免的 UX 反模式：**
从分析您领域的成功和失败，以下是要避免的模式：

- [反模式 1] - 用户觉得这令人困惑/沮丧
- [反模式 2] - 这会造成不必要的摩擦
- [反模式 3] - 与您的 [情感目标] 不一致

从他人错误中学习与从成功中学习同样重要。"

### 5. 定义设计灵感策略

为使用此灵感创建清晰策略：
"**设计灵感策略：**

**要采纳的内容：**

- [特定模式] - 因为它支持 [您的核心体验]
- [特定模式] - 因为它与 [用户需求] 一致

**要调整的内容：**

- [特定模式] - 为 [您的独特需求] 进行修改
- [特定模式] - 为 [您的用户技能水平] 进行简化

**要避免的内容：**

- [特定反模式] - 与 [您的目标] 冲突
- [特定反模式] - 不适合 [您的平台]

此策略将指导我们的设计决策，同时保持 {{project_name}} 的独特性。"
