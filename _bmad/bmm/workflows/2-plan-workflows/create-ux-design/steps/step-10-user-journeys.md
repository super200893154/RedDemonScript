## 用户旅程流程序列：

### 1. 加载 PRD 用户旅程作为基础

从 PRD 中已定义的用户旅程开始：
"太好了！既然我们已有 PRD，让我们基于其中已记录的用户旅程继续。

**PRD 中的现有用户旅程：**
我已从您的 PRD 加载了这些用户旅程：
[PRD 输入文档中的旅程叙述]

这些旅程告诉我们用户是**谁**以及他们**为什么**采取某些行动。现在我们需要设计这些旅程具体**如何**运作。

**需要设计流程的关键旅程：**
查看 PRD 旅程，我需要为以下内容设计详细的交互流程：

- [从 PRD 叙述识别的关键旅程 1]
- [从 PRD 叙述识别的关键旅程 2]
- [从 PRD 叙述识别的关键旅程 3]

PRD 给了我们故事 - 现在我们设计机制！"

### 2. 设计每个旅程流程

为每个关键旅程设计详细流程：

**对于 [旅程名称]：**
"让我们为完成 [旅程目标] 的用户设计流程。

**流程设计问题：**

- 用户如何开始此旅程？（入口点）
- 他们在每一步需要什么信息？
- 他们需要做出什么决策？
- 他们如何知道自己正在成功推进？
- 此旅程的成功是什么样的？
- 他们可能在何处感到困惑或卡住？
- 他们如何从错误中恢复？"

### 3. 创建流程图

使用 Mermaid 图表可视化每个旅程：
"我将为每个旅程创建详细的流程图，展示：

**[旅程名称] 流程：**

- 入口点和触发器
- 决策点和分支
- 成功和失败路径
- 错误恢复机制
- 信息的渐进式披露

每个图将映射从开始到结束的完整用户体验。"

### 4. 优化效率和愉悦

完善流程以获得最佳用户体验：
"**流程优化：**
对于每个旅程，让我们确保：

- 最小化到达价值的步骤（让用户快速达成成功）
- 减少每个决策点的认知负担
- 提供清晰的反馈和进度指示器
- 创造愉悦或成就的时刻
- 优雅处理边缘情况和错误恢复

**具体优化：**

- [旅程效率优化 1]
- [用户愉悦优化 2]
- [错误处理优化 3]"

### 5. 记录旅程模式

提取跨旅程的可复用模式：
"**旅程模式：**
在这些流程中，我看到一些可以标准化的常见模式：

**导航模式：**

- [导航模式 1]
- [导航模式 2]

**决策模式：**

- [决策模式 1]
- [决策模式 2]

**反馈模式：**

- [反馈模式 1]
- [反馈模式 2]

这些模式将确保所有用户体验的一致性。"
