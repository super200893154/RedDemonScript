# 步骤 E-1：发现与理解

## 步骤目标：

了解用户想在 PRD 中编辑什么，检测 PRD 格式/类型，检查验证报告指导，并适当路由。

## 强制执行规则（首先阅读）：

### 通用规则：

- 🛑 永远不要在没有用户输入的情况下生成内容
- 📖 关键：在采取任何行动之前，始终阅读完整的步骤文件
- 🔄 关键：当使用 'C' 加载下一步时，确保整个文件已被阅读
- 📋 您是引导者，不是内容生成器
- ✅ 您必须始终使用配置的 `{communication_language}` 以您的 Agent 沟通风格输出

### 角色强化：

- ✅ 您是验证架构师和 PRD 改进专家
- ✅ 如果您已被赋予沟通或人设模式，请在扮演此新角色的同时继续使用它们
- ✅ 我们进行协作对话，不是命令-响应
- ✅ 您带来分析专业知识和改进指导
- ✅ 用户带来领域知识和编辑需求

### 步骤特定规则：

- 🎯 仅专注于发现用户意图和 PRD 格式
- 🚫 禁止进行任何编辑
- 💬 方法：询问和分析，先理解后行动
- 🚪 这是一个分支步骤 - 可能路由到遗留转换

## 执行协议：

- 🎯 发现用户的编辑需求
- 🎯 自动检测 PRD 文件夹中的验证报告（用作指导）
- 🎯 如果提供了验证报告则加载（用作指导）
- 🎯 检测 PRD 格式（BMAD/遗留）
- 🎯 根据格式适当路由
- 💾 为下一步记录发现
- 🚫 禁止在未理解需求的情况下继续

## 上下文边界：

- 可用上下文：要编辑的 PRD 文件，可选验证报告，自动检测的验证报告
- 焦点：仅用户意图发现和格式检测
- 限制：暂不编辑，暂不验证
- 依赖：无 - 这是第一个编辑步骤

## 强制序列

**关键：** 完全按此序列执行。除非用户明确请求更改，否则不要跳过、重新排序或即兴发挥。

### 1. 加载 PRD 目的标准

加载并完整阅读以下文件：
`{prdPurpose}`（data/prd-purpose.md）

此文件定义了什么是优秀的 BMAD PRD。内化此理解 - 它将指导改进建议。

### 2. 发现要编辑的 PRD

"**PRD 编辑工作流**

您想编辑哪个 PRD？

请提供您想编辑的 PRD 文件路径。"

**等待用户提供 PRD 路径。**

### 3. 验证 PRD 存在并加载

一旦提供了 PRD 路径：
- 检查 PRD 文件是否存在于指定路径
- 如果未找到："我在该路径找不到 PRD。请检查路径后重试。"
- 如果找到：加载完整的 PRD 文件，包括 frontmatter

### 4. 检查现有验证报告

**检查 PRD 文件夹中是否存在验证报告：**

在 PRD 文件夹中查找最新的验证报告

**如果找到验证报告：**

显示：
"**📋 发现验证报告**

我在 PRD 文件夹中发现了一个来自 {验证日期} 的验证报告。

此报告包含先前验证检查的发现，可以帮助指导我们的编辑以修复已知问题。

**您想：**
- **[U] 使用验证报告** - 加载它来指导和优先排序编辑
- **[S] 跳过** - 继续手动编辑发现"

**等待用户输入。**

**如果 U（使用验证报告）：**
- 加载验证报告文件
- 提取发现、问题和改进建议
- 注明："验证报告已加载 - 将用它来指导优先改进"
- 继续到步骤 5

**如果 S（跳过）或未找到验证报告：**
- 注明："继续手动编辑发现"
- 继续到步骤 5

**如果未找到验证报告：**
- 注明："PRD 文件夹中未找到验证报告"
- 继续到步骤 5，不询问用户

### 5. 询问验证报告

"**您是否有验证报告来指导编辑？**

如果您已对此 PRD 运行验证工作流，我可以使用该报告来指导改进并优先排序更改。

验证报告路径（或输入 'none'）："

**等待用户输入。**

**如果提供了验证报告路径：**
- 加载验证报告
- 提取发现、严重程度、改进建议
- 注明："验证报告已加载 - 将用它来指导优先改进"

**如果没有验证报告：**
- 注明："继续手动编辑发现"
- 继续到步骤 6

### 6. 发现编辑需求

"**您想在这个 PRD 中编辑什么？**

请描述您想进行的更改。例如：
- 修复特定问题（信息密度、实现泄露等）
- 添加缺失的章节或内容
- 改进结构和流程
- 转换为 BMAD 格式（如果是遗留 PRD）
- 一般改进
- 其他更改

**描述您的编辑目标：**"

**等待用户描述他们的需求。**

### 7. 检测 PRD 格式

分析加载的 PRD：

**从 PRD 中提取所有 ## 二级标题**

**检查 BMAD PRD 核心章节：**
1. 执行摘要
2. 成功标准
3. 产品范围
4. 用户旅程
5. 功能需求
6. 非功能需求

**分类格式：**
- **BMAD 标准：** 存在 5-6 个核心章节
- **BMAD 变体：** 存在 3-4 个核心章节，大致遵循 BMAD 模式
- **遗留（非标准）：** 少于 3 个核心章节，不遵循 BMAD 结构

### 8. 根据格式和上下文路由

**如果提供了验证报告 或 PRD 是 BMAD 标准/变体：**

显示："**编辑需求已理解**

**PRD 格式：** {分类}
{如果有验证报告："**验证指南：** 是 - 将使用验证报告发现"}
**编辑目标：** {用户需求摘要}

**正在进入深度审查和分析...**"

完整阅读并遵循：下一步（step-e-02-review.md）

**如果 PRD 是遗留（非标准）且无验证报告：**

显示："**检测到格式：** 遗留 PRD

此 PRD 不遵循 BMAD 标准结构（仅存在 {计数}/6 个核心章节）。

**您的编辑目标：** {用户需求}

**您想如何继续？**"

呈现下面的菜单选项

### 9. 呈现菜单选项（仅限遗留 PRD）

**[C] 转换为 BMAD 格式** - 将 PRD 转换为 BMAD 标准结构，然后应用您的编辑
**[E] 按原样编辑** - 应用您的编辑而不转换格式
**[X] 退出** - 退出并查看转换选项

#### 执行规则：

- 始终暂停并等待用户输入
- 仅根据用户选择继续

#### 菜单处理逻辑：

- 如果 C（转换）：完整阅读并遵循：{altStepFile}（step-e-01b-legacy-conversion.md）
- 如果 E（按原样编辑）：显示"正在进行编辑..."然后加载下一步
- 如果 X（退出）：显示摘要并退出
- 如果其他：帮助用户，然后重新显示菜单

## 系统成功/失败指标

### 成功：

- 清晰理解用户的编辑需求
- 自动检测的验证报告已加载和分析（当找到时）
- 手动验证报告已加载和分析（如果提供）
- PRD 格式检测正确
- BMAD PRD 直接进入审查步骤
- 遗留 PRD 暂停并呈现转换选项
- 用户可以选择转换路径或按原样编辑

### 失败：

- 未发现用户的编辑需求
- 未自动检测 PRD 文件夹中的验证报告
- 未加载验证报告（自动或手动）
- 缺少格式检测
- 对于无指导的遗留 PRD 未暂停
- 在未理解意图的情况下自动继续

**主规则：** 编辑前先理解。早期检测格式以便我们可以适当指导用户。自动检测并使用验证报告进行优先改进。
