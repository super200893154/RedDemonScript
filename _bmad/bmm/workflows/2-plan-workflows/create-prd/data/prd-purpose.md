# PRD 目的选项

此文件包含在 PRD 初始化期间选择的目的陈述。目的指导 PRD 内容并确保关注正确的目标。

## 目的陈述类别

### 新产品开发
用于创建新解决方案的 PRD。

#### 完整产品发布
"为全新产品创建完整的 PRD，定义核心功能、用户故事、技术需求和成功指标。"

#### MVP/原型
"创建专注于核心价值主张和基本功能的最小可行产品 PRD。"

#### 产品线扩展
"为扩展现有产品线的新产品创建 PRD，利用既定的架构和用户基础。"

### 产品演进
用于改进或扩展现有产品的 PRD。

#### 功能增强
"为向现有产品添加重要功能创建 PRD，定义新功能及其与现有功能的集成。"

#### 用户体验改进
"为专注于用户体验改进、可用性增强和界面优化的 PRD 创建。"

#### 性能优化
"为解决性能瓶颈、可扩展性改进和技术优化的 PRD 创建。"

### 平台与技术
用于平台级变化的 PRD。

#### 平台迁移
"为将产品迁移到新平台、技术栈或基础设施创建 PRD。"

#### 技术重构
"为现代化技术架构同时保持现有功能的 PRD 创建。"

#### 集成项目
"为与第三方服务、API 或其他系统集成创建 PRD。"

### 业务与运营
用于业务驱动的 PRD。

#### 合规与监管
"为满足监管要求、合规标准或法律义务创建 PRD。"

#### 市场扩展
"为进入新市场、区域或细分市场的产品适配创建 PRD。"

#### 运营效率
"为改进运营流程、自动化工作流或减少人工投入的 PRD 创建。"

### 维护与质量
用于产品维护的 PRD。

#### 缺陷修复与稳定性
"为解决关键缺陷、提高稳定性和修复产品质量问题的 PRD 创建。"

#### 技术债务
"为减少技术债务、改进代码质量和增强可维护性创建 PRD。"

#### 安全更新
"为安全增强、漏洞修复和安全合规创建 PRD。"

## 目的选择指南

### 何时选择每个目的

| 目的 | 最适合 | 关键重点 |
|------|--------|----------|
| 完整产品发布 | 全新产品 | 全面功能集、市场定位 |
| MVP/原型 | 早期验证 | 核心价值主张、快速交付 |
| 产品线扩展 | 利用品牌 | 与现有产品一致、新功能 |
| 功能增强 | 增值 | 与现有功能集成、用户价值 |
| UX 改进 | 用户满意度 | 可用性、可访问性、满意度 |
| 性能优化 | 可扩展性 | 速度、效率、资源使用 |
| 平台迁移 | 现代化 | 功能对等、迁移路径 |
| 技术重构 | 技术健康 | 架构、可维护性 |
| 集成项目 | 生态系统 | API 设计、数据流 |
| 合规/监管 | 风险管理 | 要求、时间表、审计 |
| 市场扩展 | 增长 | 本地化、文化适配 |
| 运营效率 | 成本减少 | 自动化、流程改进 |
| 缺陷修复 | 质量 | 根因、测试覆盖 |
| 技术债务 | 可持续性 | 重构优先级、影响分析 |
| 安全更新 | 风险缓解 | 漏洞评估、安全测试 |

### 目的组合

有些 PRD 可能服务于多个目的。优先考虑：

1. **主要目的**：驱动 PRD 的主要目标
2. **次要目的**：支持性目标，在主要目标解决后处理

### 目的对 PRD 内容的影响

所选目的影响：

- **目标受众**：谁会阅读此 PRD
- **详细程度**：深度 vs 广度
- **重点领域**：哪些部分需要最多关注
- **成功指标**：如何衡量成功
- **时间表考虑**：现实的交付预期
- **风险评估**：哪些风险最相关

---

**注意**：PRD 工作流期间，所选目的将记录在 PRD frontmatter 中的 `prdPurpose` 字段，并用于指导内容生成决策。
