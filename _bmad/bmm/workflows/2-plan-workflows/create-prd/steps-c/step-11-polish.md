## 您的任务：

优化完整 PRD 文档的流程、连贯性和专业呈现，同时保留所有必要信息。

## 文档润色序列：

### 1. 加载上下文和文档

**关键：** 首先加载 PRD 目的文档：

- 阅读 `{purposeFile}` 了解什么是优秀的 BMAD PRD
- 内化理念：信息密度、可追溯性、可衡量需求
- 记住双重受众性质（人类 + LLM）

**然后加载 PRD 文档：**

- 从头到尾完整阅读 `{outputFile}`
- 理解完整文档结构和内容
- 识别所有章节及其关系
- 注意需要注意的领域

### 2. 文档质量审查

根据 PRD 目的原则审查整个文档：

**信息密度：**
- 是否有可以压缩的冗长短语？
- 是否存在对话填充词？
- 句子是否可以更直接和简洁？

**流程和连贯性：**
- 章节之间过渡是否流畅？
- 是否有突兀的主题转换？
- 文档是否讲述连贯的故事？
- 进展对读者是否合乎逻辑？

**重复检测：**
- 想法是否在多个章节中重复？
- 相同信息是否陈述多次？
- 冗余内容是否可以整合？
- 是否存在矛盾的陈述？

**标题结构：**
- 所有主要章节是否使用 ## 二级标题？
- 层次结构是否一致（##、###、####）？
- 章节是否可以轻松提取或引用？
- 标题是否具有描述性和清晰？

**可读性：**
- 句子是否清晰简洁？
- 语言在整个文档中是否一致？
- 技术术语是否使用得当？
- 利益相关者是否会认为这易于理解？

### 3. 优化行动

进行有针对性的改进：

**改善流程：**
- 在章节之间添加过渡句子
- 平滑突兀的主题转换
- 确保逻辑进展
- 跨章节连接相关概念

**减少重复：**
- 整合重复信息
- 将内容保留在最合适的章节
- 使用交叉引用代替重复
- 删除冗余解释

**增强连贯性：**
- 确保整个文档术语一致
- 将所有章节与产品差异化因素对齐
- 保持一致的语气和风格
- 验证跨章节范围一致性

**优化标题：**
- 确保所有主要章节使用 ## 二级标题
- 使标题具有描述性和行动导向
- 检查标题是否遵循一致模式
- 验证标题支持文档导航

### 4. 保留关键信息

**在优化时，确保没有重要内容丢失：**

**必须保留：**
- 所有用户成功标准
- 所有功能需求（能力契约）
- 所有用户旅程叙述
- 所有范围决策（MVP、增长、愿景）
- 所有非功能需求
- 产品差异化因素和愿景
- 领域特定需求
- 创新分析（如存在）

**可以整合：**
- 对同一概念的重复解释
- 冗余背景信息
- 类似内容的多个版本
- 重叠的示例

### 5. 生成优化文档

创建润色版本：

**润色过程：**
1. 从原始文档开始
2. 应用所有优化行动
3. 审查确保没有重要内容丢失
4. 验证改进增强可读性
5. 准备优化版本以供审查

### 6. 呈现菜单选项

呈现润色文档以供审查，然后显示菜单：
- 显示润色中更改的内容
- 突出所做的改进（流程、重复、标题）
- 询问他们是否想进一步细化、获取其他观点或继续
- 作为对话的一部分自然地呈现菜单选项

显示："**选择：** [A] 高级引导 [P] 派对模式 [C] 继续到完成 PRD（第 12 步，共 12 步）"

#### 菜单处理逻辑：

- 如果 P：完整阅读并遵循：{partyModeWorkflow} 使用润色文档，处理对流程和连贯性的协作细化，询问用户"接受这些润色更改？(y/n)"，如果是则用改进更新内容然后重新显示菜单，如果否则保留原始润色然后重新显示菜单
