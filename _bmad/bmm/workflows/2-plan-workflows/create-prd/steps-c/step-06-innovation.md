## 您的任务：

检测和探索产品中的创新模式，专注于其真正新颖的内容以及如何验证创新方面。

## 创新发现序列：

### 1. 加载项目类型创新数据

加载特定于该项目类型的创新信号：

- 完整加载 `{projectTypesCSV}`
- 找到 `project_type` 匹配步骤 02 检测到的类型的行
- 提取 `innovation_signals`（分号分隔列表）
- 提取 `web_search_triggers` 用于潜在的创新研究

### 2. 倾听创新指标

监控对话中的通用和项目类型特定的创新信号：

#### 通用创新语言：

- "没有类似的东西存在"
- "我们正在重新思考 [X] 如何工作"
- "首次将 [A] 与 [B] 结合"
- "对 [问题] 的新颖方法"
- "以前没有人做过 [概念]"

#### 项目类型特定信号（来自 CSV）：

将用户描述与其 project_type 的 innovation_signals 匹配：

- **api_backend**："API 组合；新协议"
- **mobile_app**："手势创新；AR/VR 功能"
- **saas_b2b**："工作流自动化；AI 代理"
- **developer_tool**："新范式；DSL 创建"

### 3. 初始创新筛查

询问有针对性的创新发现问题：
- 引导探索产品创新的内容
- 探索他们是否在挑战现有假设
- 询问技术/方法的新颖组合
- 识别以前未做过什么
- 了解哪些方面感觉最创新

### 4. 深度创新探索（如果检测到）

如果发现创新信号，深入探索：

#### 创新发现问题：
- 与现有解决方案相比，它有什么独特之处？
- 您在挑战什么假设？
- 我们如何验证它有效？
- 如果无效，备选方案是什么？
- 以前有人尝试过吗？

#### 市场背景研究：

如果检测到相关创新，考虑网络搜索获取背景：
使用项目类型 CSV 中的 `web_search_triggers`：
`[web_search_triggers] {概念} 创新 {日期}`

### 5. 生成创新内容（如果检测到创新）

准备要追加到文档的内容：

#### 内容结构：

保存到文档时，追加这些二级和三级章节：

```markdown
## 创新与新颖模式

### 检测到的创新领域

[基于对话识别的创新模式]

### 市场背景与竞争格局

[基于对话的市场背景和研究]

### 验证方法

[基于对话的验证方法论]

### 风险缓解

[基于对话的创新风险和备选方案]
```

### 6. 呈现菜单选项（仅当检测到创新时）

呈现创新内容以供审查，然后显示菜单：
- 显示识别的创新方面（使用第 5 节的结构）
- 突出与现有解决方案的差异化
- 询问他们是否想进一步细化、获取其他观点或继续
- 作为对话的一部分自然地呈现菜单选项

显示："**选择：** [A] 高级引导 [P] 派对模式 [C] 继续到项目类型分析（第 7 步，共 11 步）"

#### 菜单处理逻辑：
- 如果 A：完整阅读并遵循：{advancedElicitationTask} 使用当前创新内容，处理返回的增强创新洞察，询问用户"接受这些对创新分析的改进？(y/n)"，如果是则用改进更新内容然后重新显示菜单，如果否则保留原始内容然后重新显示菜单
- 如果 P：完整阅读并遵循：{partyModeWorkflow} 使用当前创新内容，处理协作创新探索和构思，询问用户"接受这些对创新分析的更改？(y/n)"，如果是则用改进更新内容然后重新显示菜单，如果否则保留原始内容然后重新显示菜单
- 如果 C：将最终内容追加到 {outputFile}，更新 frontmatter 将此步骤名称添加到 stepsCompleted 数组末尾，然后完整阅读并遵循：{nextStepFile}
- 如果其他：帮助用户响应，然后重新显示菜单

#### 执行规则：

- 仅当用户选择 'C' 时才继续下一步
- 执行其他菜单项后，返回此菜单

## 未检测到创新：

如果探索后未发现真正的创新信号：
- 确认未找到清晰的创新信号
- 说明这没问题 - 许多成功的产品是现有概念的优秀执行
- 询问他们是否想尝试寻找创新角度或继续

显示："**选择：** [A] 高级引导 - 让我们尝试寻找创新角度 [C] 继续 - 跳过创新章节并进入项目类型分析（第 7 步，共 11 步）"
