### 1. 尝试子流程验证

**尝试使用 Task 工具启动子流程：**

"对此 PRD 执行可追溯性验证：

1. 从执行摘要提取内容（愿景、目标）
2. 提取成功标准
3. 提取用户旅程（用户类型、流程、结果）
4. 提取功能需求（FR）
5. 提取产品范围（范围内项目）

**验证链：**
- 执行摘要 → 成功标准：愿景是否与定义的成功一致？
- 成功标准 → 用户旅程：成功标准是否由用户旅程支持？
- 用户旅程 → 功能需求：每个 FR 是否可以追溯到用户旅程？
- 范围 → FR：MVP 范围 FR 是否与范围内项目一致？

**识别孤立项：**
- 无法追溯到任何用户旅程或业务目标的 FR
- 没有用户旅程支持的成功标准
- 没有支持 FR 的用户旅程

构建可追溯矩阵并识别断裂链和孤立 FR。

返回带有链状态和孤立列表的结构化发现。"

### 2. 优雅降级（如果 Task 工具不可用）

如果 Task 工具不可用，直接执行分析：

**步骤 1：提取关键元素**
- 执行摘要：注意愿景、目标、目的
- 成功标准：列出所有标准
- 用户旅程：列出用户类型及其流程
- 功能需求：列出所有 FR
- 产品范围：列出范围内项目

**步骤 2：验证执行摘要 → 成功标准**
- 执行摘要是否提到成功维度？
- 成功标准是否与愿景一致？
- 注意任何不一致

**步骤 3：验证成功标准 → 用户旅程**
- 对于每个成功标准，是否有用户旅程实现它？
- 注意没有支持旅程的成功标准

**步骤 4：验证用户旅程 → FR**
- 对于每个用户旅程/流程，是否有 FR 支持它？
- 列出没有明确用户旅程来源的 FR
- 注意孤立 FR（没有可追溯来源的需求）

**步骤 5：验证范围 → FR 一致性**
- MVP 范围是否与基本 FR 一致？
- 范围内项目是否由 FR 支持？
- 注意不一致

**步骤 6：构建可追溯矩阵**
- 将每个 FR 映射到其来源（旅程或业务目标）
- 注意孤立 FR
- 识别断裂链

### 3. 统计可追溯性问题

**断裂链：**
- 执行摘要 → 成功标准差距：计数
- 成功标准 → 用户旅程差距：计数
- 用户旅程 → FR 差距：计数
- 范围 → FR 不一致：计数

**孤立元素：**
- 孤立 FR（无可追溯来源）：计数
- 无支持的成功标准：计数
- 无 FR 的用户旅程：计数

**总问题数：** 所有断裂链和孤立项之和
