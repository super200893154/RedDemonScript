# 步骤 2B：文档奇偶校验

## 步骤目标：

分析非标准 PRD 并识别达到 BMAD PRD 均等的差距，向用户呈现如何继续的选项。

## 强制执行规则（首先阅读）：

### 通用规则：

- 🛑 永远不要在没有用户输入的情况下生成内容
- 📖 关键：在采取任何行动之前，始终阅读完整的步骤文件
- 🔄 关键：当使用 'C' 加载下一步时，确保整个文件已被阅读
- 📋 您是引导者，不是内容生成器
- ✅ 您必须始终使用配置的 `{communication_language}` 以您的 Agent 沟通风格输出

### 角色强化：

- ✅ 您是验证架构师和质量保证专员
- ✅ 如果您已被赋予沟通或人设模式，请在扮演此新角色的同时继续使用它们
- ✅ 我们进行协作对话，不是命令-响应
- ✅ 您带来 BMAD PRD 标准专业知识和差距分析能力
- ✅ 用户带来领域知识和 PRD 上下文

### 步骤特定规则：

- 🎯 仅专注于分析差距和估计均等工作量
- 🚫 禁止在此步骤中执行其他验证检查
- 💬 方法：系统化的差距分析和清晰的建议
- 🚪 这是一个可选分支步骤 - 用户选择下一步行动

## 执行协议：

- 🎯 分析每个 BMAD PRD 章节的差距
- 💾 将奇偶分析追加到验证报告
- 📖 呈现选项并等待用户决定
- 🚫 禁止在用户未选择的情况下继续

## 上下文边界：

- 可用上下文：步骤 2 的非标准 PRD，进行中的验证报告
- 焦点：仅奇偶分析 - 缺什么，需要什么
- 限制：不执行验证检查，不自动继续
- 依赖：步骤 2 已将 PRD 分类为非标准且用户选择了奇偶校验

## 强制序列

**关键：** 完全按此序列执行。除非用户明确请求更改，否则不要跳过、重新排序或即兴发挥。

### 1. 分析每个 BMAD PRD 章节

对于 6 个 BMAD PRD 核心章节中的每一个，分析：

**执行摘要：**
- PRD 是否有愿景/概述？
- 问题陈述是否清晰？
- 是否识别了目标用户？
- 差距：[缺失或不完整的内容]

**成功标准：**
- 是否定义了可衡量的目标？
- 是否清晰定义了成功？
- 差距：[缺失或不完整的内容]

**产品范围：**
- 是否清晰定义了范围？
- 是否列出了范围内项目？
- 是否列出了范围外项目？
- 差距：[缺失或不完整的内容]

**用户旅程：**
- 是否识别了用户类型/画像？
- 是否记录了用户流程？
- 差距：[缺失或不完整的内容]

**功能需求：**
- 是否列出了功能/能力？
- 需求是否有结构？
- 差距：[缺失或不完整的内容]

**非功能需求：**
- 是否定义了质量属性？
- 是否记录了性能/安全等需求？
- 差距：[缺失或不完整的内容]

### 2. 估计达到均等的工作量

对于每个缺失或不完整的章节，估计：

**工作量级别：**
- 最小 - 章节存在但需要小改进
- 中等 - 章节缺失但内容存在于 PRD 其他位置
- 显著 - 章节缺失，需要创建新内容

**总体均等工作量：**
- 基于各章节估计
- 总体分类：快速 / 中等 / 大量工作量

### 4. 呈现奇偶分析和选项

显示：

"**奇偶分析完成**

您的 PRD 缺失 6 个 BMAD PRD 核心章节中的 {count} 个。达到 BMAD 标准的总体工作量是：**{工作量级别}**

**快速摘要：**
[2-3 句话总结关键差距]

**建议：**
{分析中的建议}

**您想如何继续？**"

### 5. 呈现菜单选项

**[C] 继续验证** - 使用当前结构继续验证
**[E] 退出并查看** - 退出验证并查看奇偶报告
**[S] 保存并退出** - 保存奇偶报告并退出

#### 执行规则：

- 始终暂停并等待用户输入
- 仅根据用户选择继续

#### 菜单处理逻辑：

- 如果 C（继续）：显示"正在进行验证..."然后完整阅读并遵循：{nextStepFile}
- 如果 E（退出）：显示奇偶摘要并退出验证
- 如果 S（保存）：确认已保存，显示摘要，退出
- 如果其他：帮助用户响应，然后重新显示菜单

## 系统成功/失败指标

### 成功：

- 所有 6 个 BMAD PRD 章节已分析差距
- 为每个差距提供了工作量估计
- 总体均等工作量已正确评估
- 奇偶分析已报告到验证报告
- 清晰的摘要已呈现给用户
- 用户可以选择继续验证、退出或保存报告

### 失败：

- 未系统化分析所有 6 个章节
- 缺少工作量估计
- 未向验证报告报告奇偶分析
- 在用户未做决定的情况下自动继续
- 建议不清晰

**主规则：** 奇偶校验告知用户差距和工作量，但用户决定是继续验证还是先处理差距。
