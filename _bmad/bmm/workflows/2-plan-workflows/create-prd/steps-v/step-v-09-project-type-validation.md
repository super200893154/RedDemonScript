### 1. 加载项目类型数据

加载并完整阅读以下文件：
`{projectTypesData}`（../data/project-types.csv）

此 CSV 包含：
- 每种项目类型的检测信号
- 每种项目类型的必需章节
- 每种项目类型的跳过/排除章节
- 创新信号

内化此数据 - 它决定每种项目类型必须存在或排除哪些章节。

### 2. 提取项目类型分类

从 PRD frontmatter 中提取：
- `classification.projectType` - 这是什么类型的项目？

**常见项目类型：**
- api_backend
- web_app
- mobile_app
- desktop_app
- data_pipeline
- ml_system
- library_sdk
- infrastructure
- other

**如果未找到 projectType 分类：**
假设为"web_app"（最常见）并在发现中注明

### 3. 从 CSV 数据确定必需和排除章节

**从加载的 project-types.csv 数据，对于此项目类型：**

**必需章节：**（来自 required_sections 列）
这些必须存在于 PRD 中

**跳过章节：**（来自 skip_sections 列）
这些不能存在于 PRD 中

**CSV 中的示例映射：**
- api_backend：必需=[endpoint_specs, auth_model, data_schemas]，跳过=[ux_ui, visual_design]
- mobile_app：必需=[platform_reqs, device_permissions, offline_mode]，跳过=[desktop_features, cli_commands]
- cli_tool：必需=[command_structure, output_formats, config_schema]，跳过=[visual_design, ux_principles, touch_interactions]
- 等等。

### 4. 基于 CSV 需求验证

**基于项目类型，确定：**

**api_backend：**
- 必需：端点规范、认证模型、数据模式、API 版本控制
- 排除：UX/UI 章节、移动特定章节

**web_app：**
- 必需：用户旅程、UX/UI 需求、响应式设计
- 排除：通常无

**mobile_app：**
- 必需：移动 UX、平台特性（iOS/Android）、离线模式
- 排除：桌面特定章节

**desktop_app：**
- 必需：桌面 UX、平台特性（Windows/Mac/Linux）
- 排除：移动特定章节

**data_pipeline：**
- 必需：数据源、数据转换、数据目标、错误处理
- 排除：UX/UI 章节

**ml_system：**
- 必需：模型需求、训练数据、推理需求、模型性能
- 排除：UX/UI 章节（除非 ML UI）

**library_sdk：**
- 必需：API 接口、使用示例、集成指南
- 排除：UX/UI 章节、部署章节

**infrastructure：**
- 必需：基础设施组件、部署、监控、扩展
- 排除：功能需求（这是基础设施，不是产品）

### 4. 尝试子流程验证

"对 {项目类型} 执行项目类型合规验证：

**检查必需章节是否存在：**
{列出此项目类型的必需章节}
对于每个：PRD 中是否存在？文档是否充分？

**检查排除章节是否不存在：**
{列出此项目类型的排除章节}
对于每个：PRD 中是否不存在？（应该不存在）

构建合规表显示：
- 必需章节：[存在/缺失/不完整]
- 排除章节：[不存在/存在]（存在 = 违规）

返回带有发现的合规表。"

**优雅降级（如果没有 Task 工具）：**
- 手动检查 PRD 中的必需章节
- 手动检查 PRD 中的排除章节
- 构建合规表

### 5. 构建合规表

**必需章节检查：**
- 对于每个必需章节：存在 / 缺失 / 不完整
- 计数：存在的必需章节 vs 总必需章节

**排除章节检查：**
- 对于每个排除章节：不存在 / 存在（违规）
- 计数：存在的排除章节（违规）

**总合规分数：**
- 必需：{存在}/{总计}
- 排除违规：{计数}

```markdown
{如果缺失或不完整：注意具体差距}

[继续所有必需章节]

### 排除章节（不应存在）

**{章节 1}：** [不存在/存在] ✓
{如果存在：此章节不应存在于 {项目类型} 中}

**{章节 2}：** [不存在/存在] ✓
{如果存在：此章节不应存在于 {项目类型} 中}

[继续所有排除章节]

### 合规摘要

**必需章节：** {存在}/{总计} 存在
**存在的排除章节：** {违规}（应为 0）
**合规分数：** {百分比}%

**严重程度：** [如果缺失必需章节为严重，如果不完整为警告，如果完整为通过]

**建议：**
[如果严重] "PRD 缺失 {项目类型} 的必需章节。添加缺失章节以正确指定此类型的项目。"
[如果警告] "一些 {项目类型} 的必需章节不完整。加强文档。"
[如果通过] "{项目类型} 的所有必需章节都存在。未发现排除章节。"
```
