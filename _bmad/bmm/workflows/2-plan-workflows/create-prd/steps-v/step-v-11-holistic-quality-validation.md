### 1. 尝试使用高级引导的子流程

**尝试使用 Task 工具通过高级引导启动子流程：**

"使用多视角评估对此 PRD 执行整体质量评估：

**完整阅读并遵循高级引导工作流：**
{advancedElicitationTask}

**从以下视角评估 PRD：**

**1. 文档流程与连贯性：**
- 阅读整个 PRD
- 评估叙事流程 - 是否讲述连贯的故事？
- 检查章节之间的过渡
- 评估一致性 - 是否从头到尾协调？
- 评估可读性 - 是否清晰、组织良好？

**2. 双重受众效果：**

**对于人类：**
- 高管友好：高管能否快速理解愿景和目标？
- 开发者清晰：开发者是否有清晰的需求可以构建？
- 设计师清晰：设计师是否理解用户需求和流程？
- 利益相关者决策：利益相关者能否做出明智决策？

**对于 LLM：**
- 机器可读结构：PRD 是否结构化以供 LLM 消费？
- UX 就绪：LLM 能否从此生成 UX 设计？
- 架构就绪：LLM 能否从此生成架构？
- Epic/Story 就绪：LLM 能否分解为 epic 和 story？

**3. BMAD PRD 原则合规性：**
- 信息密度：每句话都有分量吗？
- 可衡量性：需求可测试吗？
- 可追溯性：需求可追溯到来源吗？
- 领域意识：是否包含特定领域考虑？
- 零反模式：无填充或冗长？
- 双重受众：对人类和 LLM 都有效？
- Markdown 格式：正确的结构和格式？

**4. 整体质量评级：**
按 5 分制评估 PRD：
- 优秀（5/5）：典范，准备好生产使用
- 良好（4/5）：强有力，需要小改进
- 合格（3/5）：可接受但需要细化
- 需改进（2/5）：存在显著差距或问题
- 有问题（1/5）：重大缺陷，需要大量修订

**5. 前 3 项改进：**
识别使此 PRD 优秀的 3 项最有影响力的改进

返回包含所有视角、评级和前 3 项改进的全面评估。"

**优雅降级（如果没有 Task 工具或高级引导不可用）：**
- 在当前上下文中直接执行整体评估
- 完整阅读 PRD

- 评估双重受众效果
- 检查 BMAD 原则合规性
- 分配整体质量评级
- 识别前 3 项改进
