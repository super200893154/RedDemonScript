## 强制序列

**关键：** 完全按此序列执行。除非用户明确请求更改，否则不要跳过、重新排序或即兴发挥。

### 1. 提取 PRD 结构

加载完整的 PRD 文件并提取：

**所有二级（##）标题：**
- 扫描整个 PRD 文档
- 提取所有 ## 章节标题
- 按顺序列出

**PRD frontmatter：**
- 提取 classification.domain（如果存在）
- 提取 classification.projectType（如果存在）
- 注意任何其他相关元数据

### 2. 检查 BMAD PRD 核心章节

检查 PRD 是否包含以下 BMAD PRD 核心章节：

1. **执行摘要**（或变体：## 执行摘要、## 概述、## 简介）
2. **成功标准**（或：## 成功标准、## 目标、## 目的）
3. **产品范围**（或：## 产品范围、## 范围、## 范围内、## 范围外）
4. **用户旅程**（或：## 用户旅程、## 用户故事、## 用户流程）
5. **功能需求**（或：## 功能需求、## 功能、## 能力）
6. **非功能需求**（或：## 非功能需求、## NFR、## 质量属性）

**统计匹配：**
- 这 6 个核心章节中有多少存在？
- 哪些特定章节存在？
- 哪些缺失？

### 3. 分类 PRD 格式

根据核心章节数量分类：

**BMAD 标准：**
- 存在 5-6 个核心章节
- 紧密遵循 BMAD PRD 结构

**BMAD 变体：**
- 存在 3-4 个核心章节
- 大致遵循 BMAD 模式，但可能有结构差异
- 缺少一些章节但可识别为 BMAD 风格

**非标准：**
- 少于 3 个核心章节存在
- 不遵循 BMAD PRD 结构
- 可能是完全自定义格式、遗留格式或来自其他框架

### 4. 向验证报告报告格式发现

追加到验证报告：

```markdown
## 格式检测

**PRD 结构：**
[列出所有找到的 ## 二级标题]

**存在的 BMAD 核心章节：**
- 执行摘要：[存在/缺失]
- 成功标准：[存在/缺失]
- 产品范围：[存在/缺失]
- 用户旅程：[存在/缺失]
- 功能需求：[存在/缺失]
- 非功能需求：[存在/缺失]

**格式分类：** [BMAD 标准 / BMAD 变体 / 非标准]
**存在的核心章节：** [数量]/6
```

### 5. 根据格式分类路由

**如果格式为 BMAD 标准或 BMAD 变体：**

显示："**检测到格式：** {classification}

正在进行系统化验证检查..."

立即完整阅读并遵循：{nextStepFile}（step-v-03-density-validation.md）

**如果格式为非标准（< 3 个核心章节）：**

显示："**检测到格式：** 非标准 PRD

此 PRD 不遵循 BMAD 标准结构（仅存在 {count}/6 个核心章节）。

您有以下选项："

呈现下面的菜单选项供用户选择

### 6. 呈现菜单选项（仅限非标准 PRD）

**[A] 奇偶校验** - 分析差距并估计达到 BMAD PRD 均等所需的工作量
**[B] 按原样验证** - 使用当前结构继续验证
**[C] 退出** - 退出验证并查看格式发现

#### 执行规则：

- 始终暂停并等待用户输入
- 仅根据用户选择继续

#### 菜单处理逻辑：

- 如果 A（奇偶校验）：完整阅读并遵循：{altStepFile}（step-v-02b-parity-check.md）
- 如果 B（按原样验证）：显示"正在进行验证..."然后完整阅读并遵循：{nextStepFile}
- 如果 C（退出）：显示格式发现摘要并退出验证
- 如果其他：帮助用户响应，然后重新显示菜单

## 系统成功/失败指标

### 成功：

- 所有 ## 二级标题已成功提取
- BMAD 核心章节已系统化检查
- 格式基于章节数量正确分类
- 发现已报告到验证报告
- BMAD 标准/变体 PRD 直接进入下一个验证步骤
- 非标准 PRD 暂停并向用户呈现选项
- 用户可以选择奇偶校验、按原样验证或退出

### 失败：

- 分类前未提取所有标题
- 格式分类不正确
- 未向验证报告报告发现
- 未为非标准 PRD 暂停
- 在用户未做决定的情况下为非标准格式继续

**主规则：** 格式检测决定验证路径。非标准 PRD 在继续之前需要用户选择。
