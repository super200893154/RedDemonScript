# 步骤 3：上下文完成与定稿

## 强制执行规则（首先阅读）：

- 🛑 绝不在没有用户输入的情况下生成内容
- ✅ 始终将其视为技术同伴之间的协作完成
- 📋 你是引导者，而非内容生成器
- 💬 专注于完成一个精简、针对 LLM 优化的项目上下文
- 🎯 确保所有关键规则已捕获且可执行
- ⚠️ 绝对不要时间估计 - AI 开发速度已根本改变
- ✅ 你必须始终使用配置中的 `{communication_language}` 以你的 Agent 交流风格输出

## 执行协议：

- 🎯 在采取任何行动之前展示你的分析
- 📝 审查并优化内容以提高 LLM 上下文效率
- 📖 用完成状态更新 frontmatter
- 🚫 没有更多步骤 - 这是最终步骤

## 上下文边界：

- 步骤 2 的所有规则类别已完成
- 技术栈和版本已记录
- 专注于最终审查、优化和完成
- 确保上下文文件准备好供 AI agent 使用

## 你的任务：

完成项目上下文文件，针对 LLM 效率进行优化，并提供使用和维护指导。

## 完成序列：

### 1. 审查完整的上下文文件

读取整个项目上下文文件并分析：

**内容分析：**

- 总长度和 LLM 可读性
- 规则的清晰度和具体性
- 所有关键领域的覆盖
- 每条规则的可执行性

**结构分析：**

- 部分的逻辑组织
- 格式的一致性
- 没有冗余或显而易见的信息
- 优化以便快速扫描

### 2. 针对 LLM 上下文优化

确保文件精简高效：

**内容优化：**

- 移除任何冗余规则或显而易见的信息
- 将相关规则合并为简洁的要点
- 使用具体、可执行的语言
- 确保每条规则提供独特价值

**格式优化：**

- 使用一致的 markdown 格式
- 实现清晰的章节层次结构
- 通过战略性使用粗体确保可扫描性
- 在最大化信息密度的同时保持可读性

### 3. 最终内容结构

确保最终结构遵循此优化格式：

```markdown
# AI Agent 项目上下文

_此文件包含 AI agent 在此项目实现代码时必须遵循的关键规则和模式。专注于 agent 可能会遗漏的不明显细节。_

---

## 技术栈与版本

{{简洁技术列表}}

## 关键实现规则

### 语言特定规则

{{特定语言规则}}

### 框架特定规则

{{框架模式}}

### 测试规则

{{测试要求}}

### 代码质量与风格规则

{{风格和质量模式}}

### 开发工作流规则

{{工作流模式}}

### 关键必知规则

{{反模式和边界情况}}

---

## 使用指南

**AI Agent：**

- 在实现任何代码之前阅读此文件
- 严格按照文档遵循所有规则
- 如有疑问，优先选择更严格的选项
- 如果出现新模式，更新此文件

**人类：**

- 保持此文件精简并专注于 agent 需求
- 当技术栈变化时更新
- 每季度审查过时的规则
- 随时间推移移除变得显而易见的规则

最后更新：{{date}}
```
