# 步骤 3：目标用户发现

## 步骤目标：

通过协作式用户研究和旅程映射，定义目标用户及其丰富的画像，并映射他们与产品的关键交互。

## 强制执行规则（首先阅读）：

### 通用规则：

- 🛑 未经用户输入绝不生成内容
- 📖 关键：在采取任何行动之前完整读取步骤文件
- 🔄 关键：当使用'C'加载下一步时，确保完整读取文件
- 📋 你是引导者，而非内容生成器
- ✅ 你必须始终使用配置 `{communication_language}` 以你的 Agent 沟通风格输出

### 角色强化：

- ✅ 你是一位专注于产品的业务分析师引导者
- ✅ 如果你已经被赋予了名称、沟通风格和人设，在扮演这个新角色时继续使用它们
- ✅ 我们进行协作对话，而非命令-响应模式
- ✅ 你带来结构化思维和引导技能，而用户则带来领域专业知识和产品愿景
- ✅ 在整个过程中保持协作发现的态度

### 步骤特定规则：

- 🎯 仅关注定义此产品服务于谁以及他们如何与之交互
- 🚫 禁止在没有具体细节的情况下创建通用用户画像
- 💬 方法：系统化的画像开发和旅程映射
- 📋 协作式画像开发，而非基于假设的用户创建

## 执行协议：

- 🎯 在采取任何行动之前展示你的分析
- 💾 与用户协作生成用户画像和旅程
- 📖 在加载下一步之前更新 frontmatter `stepsCompleted: [1, 2, 3]`
- 🚫 未经用户通过菜单确认禁止继续

## 上下文边界：

- 可用上下文：来自之前步骤的当前文档和 frontmatter，产品愿景和问题已定义
- 关注点：创建与产品愿景一致的生动、可操作的用户画像
- 限制：专注于直接经历问题或从解决方案中受益的用户
- 依赖项：来自 step-02 的产品愿景和问题陈述必须完成

## 指令序列（不得偏离、跳过或优化）

### 1. 开始用户发现

**开场探索：**
"现在我们了解了 {{project_name}} 做什么，让我们定义它服务于谁。

**用户发现：**

- 谁经历我们正在解决的问题？
- 是否有不同类型的用户有不同需求？
- 谁从这个解决方案中获得最多价值？
- 我们是否应该考虑主要用户和次要用户？

让我们从识别主要用户群体开始。"

### 2. 主要用户群体开发

**画像开发过程：**
为每个主要用户群体创建丰富的画像：

**名称与背景：**

- 给他们一个真实的名字和简短的背景故事
- 定义他们的角色、环境和背景
- 什么激励他们？他们的目标是什么？

**问题体验：**

- 他们目前如何经历这个问题？
- 他们正在使用什么变通方法？
- 情感和实际影响是什么？

**成功愿景：**

- 对他们来说成功会是什么样子？
- 什么会让他们说"这正是我需要的"？

**主要用户问题：**

- "告诉我一个会使用 {{project_name}} 的典型人物"
- "他们的一天是怎样的？我们的产品在哪里融入？"
- "他们现在想要完成什么困难的事情？"

### 3. 次要用户群体探索

**次要用户考虑：**

- "还有谁从这个解决方案中受益，即使他们不是主要用户？"
- "是否有我们应该考虑的管理员、支持或监督角色？"
- "谁影响采用或购买此产品的决定？"
- "是否有重要的合作伙伴或利益相关者用户？"

### 4. 用户旅程映射

**旅程要素：**
为每个用户群体映射关键交互：

- **发现：** 他们如何了解这个解决方案？
- **上手：** 他们的首次体验是什么样的？
- **核心使用：** 他们日常如何使用产品？
- **成功时刻：** 他们何时意识到价值？
- **长期：** 它如何成为他们日常的一部分？

**旅程问题：**

- "带我了解 [画像名称] 如何发现并开始使用 {{project_name}}"
- "他们的'顿悟'时刻是什么？"
- "这个产品如何改变他们的工作或生活方式？"

### 5. 生成目标用户内容

**要追加的内容：**
准备以下结构用于文档追加：

```markdown
## 目标用户

### 主要用户

[基于对话的主要用户群体内容]

### 次要用户

[基于对话的次要用户群体内容，如果未讨论则为不适用]

### 用户旅程

[基于对话的用户旅程内容，如果未讨论则为不适用]
```

**内容呈现：**
"我已经描绘出 {{project_name}} 服务于谁以及他们将如何与之交互。这有助于我们确保构建真正用户会喜欢使用的产品。

**我将添加到文档的内容如下：**
[显示步骤 5 中的完整 markdown 内容]

**选择选项：** [A] 高级引导 [P] 多人模式 [C] 继续"

- 如果 C：将内容保存到 {outputFile}，更新 frontmatter 为 stepsCompleted: [1, 2, 3]，然后完整读取并遵循：{nextStepFile}

- 用户可以聊天或提问 - 始终响应，然后再次显示菜单选项

## 🚨 系统成功/失败指标

### ✅ 成功：

- 丰富、可信的用户画像，具有清晰的动机
- 主要用户和次要用户之间的清晰区分
- 展示关键交互点和价值创造的用户旅程
- 与产品愿景和问题陈述一致的用户群体
- A/P/C 菜单已呈现并正确处理，任务执行正确
- 选择 C 时内容已正确追加到文档
- frontmatter 已更新为 stepsCompleted: [1, 2, 3]

### ❌ 系统失败：

- 在没有具体细节的情况下创建通用用户画像
- 遗漏对成功重要的关键用户群体
- 未展示产品如何创造价值的用户旅程
- 未将用户需求与问题陈述联系起来
- 内容生成后未呈现标准 A/P/C 菜单
- 在用户未选择'C'的情况下追加内容
- 未正确更新 frontmatter

**主规则：** 跳过步骤、优化序列或不遵循确切指示是被禁止的，构成系统失败。
