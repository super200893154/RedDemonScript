# 文档项目工作流 - 验证清单

## 扫描级别和可恢复性

- [ ] 为 initial_scan 和 full_rescan 模式提供扫描级别选择（quick/deep/exhaustive）
- [ ] Deep-dive 模式自动使用 exhaustive 扫描（无选择）
- [ ] Quick 扫描不读取源文件（仅模式、配置、清单）
- [ ] Deep 扫描读取项目类型关键目录中的文件
- [ ] Exhaustive 扫描读取所有源文件（排除 node_modules、dist、build）
- [ ] 工作流开始时创建状态文件（project-scan-report.json）
- [ ] 每个步骤完成后更新状态文件
- [ ] 状态文件包含架构要求的所有字段
- [ ] 如果状态文件存在且 <24 小时，显示恢复提示
- [ ] 旧状态文件（>24小时）自动归档
- [ ] 恢复功能正确加载之前的状态
- [ ] 恢复时工作流可以跳转到正确的步骤

## 即写即走架构

- [ ] 每个文档在生成后立即写入磁盘
- [ ] 写入后立即进行文档验证（部分级别）
- [ ] 写入每个文档后更新状态文件
- [ ] 写入后从上下文中清除详细发现（仅保留摘要）
- [ ] 上下文仅包含高级摘要（每部分 1-2 句话）
- [ ] 内存中不累积完整项目分析

## 批处理策略（Deep/Exhaustive 扫描）

- [ ] 对 deep 和 exhaustive 扫描级别应用批处理
- [ ] 批次按子文件夹组织（而非任意文件数）
- [ ] 大文件（>5000 LOC）使用适当判断处理
- [ ] 每个批次：读取文件、提取信息、写入输出、验证、清除上下文
- [ ] 在状态文件中跟踪批次完成（batches_completed 数组）
- [ ] 批次摘要保留在上下文中（最多 1-2 句话）

## 项目检测和分类

- [ ] 项目类型正确识别并与实际技术栈匹配
- [ ] 多部分 vs 单部分结构准确检测
- [ ] 如果是多部分，识别所有项目部分（无缺失的 client/server 等）
- [ ] 为每个部分类型加载文档要求
- [ ] 架构注册表匹配适合检测到的技术栈
