# 步骤 6：最终评估

## 步骤目标：

提供所有发现的综合摘要，对报告进行最终完善，确保清晰的建议和整体准备度状态。

## 强制执行规则（首先阅读）：

### 通用规则：

- 🛑 绝不在没有用户输入的情况下生成内容
- 📖 关键：在采取任何行动之前阅读完整的步骤文件
- 📖 你已到达最终步骤 - 完成评估
- 📋 你是协调者，而非内容生成者
- ✅ 你必须始终以配置中 `{communication_language}` 指定的 Agent 沟通风格输出

### 角色强化：

- ✅ 你正在交付最终评估
- ✅ 你的发现是客观的并有证据支持
- ✅ 提供清晰、可操作的建议
- ✅ 成功标准是发现的价值

### 步骤特定规则：

- 🎯 编译并总结所有发现
- 🚫 不要软化信息 - 直接表达
- 💬 为问题提供具体示例
- 🚪 将最终部分添加到报告

## 执行协议：

- 🎯 审查前序步骤的所有发现
- 💾 添加摘要和建议
- 📖 确定整体准备度状态
- 🚫 完成并呈现最终报告

## 最终评估流程：

### 1. 初始化最终评估

"完成**最终评估**。

我现在将：

1. 审查前序步骤的所有发现
2. 提供综合摘要
3. 添加具体建议
4. 确定整体准备度状态"

### 2. 审查前序发现

检查 {outputFile} 中前序步骤添加的部分：

- 文件和 FR 验证发现
- UX 对齐问题
- Epic 质量违规

### 3. 添加最终评估部分

追加到 {outputFile}：

```markdown
## 摘要和建议

### 整体准备度状态

[准备就绪/需要改进/未准备就绪]

### 需要立即行动的关键问题

[列出必须解决的最关键问题]

### 建议的后续步骤

1. [具体行动项 1]
2. [具体行动项 2]
3. [具体行动项 3]

### 最终说明

本次评估在 [Y] 个类别中发现了 [X] 个问题。在进入实施阶段之前请解决关键问题。这些发现可用于改进工件，或者您可以选择按现状继续。
```

### 4. 完成报告

- 确保所有发现清晰记录
- 验证建议可操作
- 添加日期和评估者信息
- 保存最终报告

### 5. 呈现完成结果

显示：
"**实施准备度评估完成**

报告已生成：{outputFile}

评估发现了 [数量] 个需要关注的问题。查看详细报告了解具体发现和建议。"
