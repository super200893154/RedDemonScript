# 步骤 3：Epic 覆盖验证

## 步骤目标：

验证 PRD 中的所有功能需求都被 epic 和 story 文档捕获，识别覆盖缺口。

## 强制执行规则（首先阅读）：

### 通用规则：

- 🛑 绝不在没有用户输入的情况下生成内容
- 📖 关键：在采取任何行动之前阅读完整的步骤文件
- 🔄 关键：当使用 'C' 加载下一步时，确保完整阅读文件
- 📋 你是协调者，而非内容生成者
- ✅ 你必须始终以配置中 `{communication_language}` 指定的 Agent 沟通风格输出

### 角色强化：

- ✅ 你是专家级产品经理和 Scrum Master
- ✅ 你的专长是需求可追溯性
- ✅ 你确保没有需求遗漏
- ✅ 成功标准是完整的 FR 覆盖

### 步骤特定规则：

- 🎯 仅专注于 FR 覆盖验证
- 🚫 不要分析 story 质量（那是后面的步骤）
- 💬 将 PRD FR 与 epic 覆盖列表进行对比
- 🚪 记录每个缺失的 FR

## 执行协议：

- 🎯 完整加载 epic 文档
- 💾 从 epic 中提取 FR 覆盖信息
- 📖 与 PRD FR 列表对比
- 🚫 禁止在未记录缺口的情况下继续

## Epic 覆盖验证流程：

### 1. 初始化覆盖验证

"开始**Epic 覆盖验证**。

我将：

1. 加载 epic 和 story 文档
2. 提取 FR 覆盖信息
3. 与上一步的 PRD FR 对比
4. 识别任何未在 epic 中覆盖的 FR"

### 2. 加载 Epic 文档

根据步骤1中的文档清单：

- 加载 epic 和 story 文档（完整版或分片版）
- 完整阅读以查找 FR 覆盖信息
- 查找类似"FR 覆盖图"的章节

### 3. 提取 Epic FR 覆盖

从 epic 文档中：

- 查找 FR 覆盖映射或列表
- 提取声称覆盖的 FR 编号
- 记录哪些 epic 覆盖哪些 FR

格式：

```
## 提取的 Epic FR 覆盖

FR1: 在 Epic X 中覆盖
FR2: 在 Epic Y 中覆盖
FR3: 在 Epic Z 中覆盖
...
Epic 中的 FR 总数：[计数]
```

### 4. 将覆盖情况与 PRD 对比

使用步骤2中的 PRD FR 列表：

- 检查每个 PRD FR 是否在 epic 覆盖中
- 识别未在 epic 中覆盖的 FR
- 记录在 epic 中但不在 PRD 中的 FR

创建覆盖矩阵：

```
## FR 覆盖分析

| FR 编号 | PRD 需求 | Epic 覆盖 | 状态 |
| ------- | -------- | --------- | ---- |
| FR1 | [PRD 文本] | Epic X Story Y | ✓ 已覆盖 |
| FR2 | [PRD 文本] | **未找到** | ❌ 缺失 |
| FR3 | [PRD 文本] | Epic Z Story A | ✓ 已覆盖 |
```

### 5. 记录缺失覆盖

列出所有未覆盖的 FR：

```
## 缺失的 FR 覆盖

### 关键缺失 FR

FR#: [来自 PRD 的完整需求文本]
- 影响：[为什么这是关键的]
- 建议：[哪个 epic 应该包含此项]

### 高优先级缺失 FR

[列出任何其他未覆盖的 FR]
```

### 6. 添加到评估报告

追加到 {outputFile}：

```markdown
## Epic 覆盖验证

### 覆盖矩阵

[第4节的完整覆盖矩阵]

### 缺失需求

[第5节未覆盖 FR 列表]

### 覆盖统计

- PRD FR 总数：[计数]
- Epic 中覆盖的 FR：[计数]
- 覆盖百分比：[百分比]
```

### 7. 自动进入下一步

覆盖验证完成后，立即加载下一步。

Epic 覆盖验证完成。加载下一步进行 UX 对齐检查。

## 🚨 系统成功/失败指标

### ✅ 成功：

- 完整加载 epic 文档
- 准确提取 FR 覆盖
- 识别并记录所有缺口
- 创建覆盖矩阵

### ❌ 系统失败：

- 未完整阅读 epic 文档
- 对比时遗漏 FR
- 未记录未覆盖的需求
- 覆盖分析不完整

**主规则：** 每个 FR 必须有可追溯的实施路径。
