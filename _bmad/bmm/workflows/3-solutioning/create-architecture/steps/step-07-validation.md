## 你的任务：

验证完整架构的一致性、完整性和指导 AI Agent 一致实施的准备度。

## 验证序列：

### 1. 一致性验证

检查所有架构决策是否协同工作：

**决策兼容性：**

- 所有技术选择是否无冲突地协同工作？
- 所有版本是否相互兼容？
- 模式是否与技术选择对齐？
- 是否有任何矛盾的决策？

**模式一致性：**

- 实施模式是否支持架构决策？
- 命名约定在所有领域是否一致？
- 结构模式是否与技术栈对齐？
- 通信模式是否连贯？

**结构对齐：**

- 项目结构是否支持所有架构决策？
- 边界是否正确定义和遵守？
- 结构是否支持选择的模式？
- 集成点是否正确结构化？

### 2. 需求覆盖验证

验证所有项目需求都有架构支持：

**从 Epic（如可用）：**

- 每个 epic 是否有架构支持？
- 所有用户故事是否可以用这些决策实施？
- 跨 epic 依赖是否在架构上处理？
- epic 覆盖是否有任何缺口？

**从 FR 类别（如无 epic）：**

- 每个功能需求是否有架构支持？
- 所有 FR 类别是否被架构决策完全覆盖？
- 横切 FR 是否正确处理？
- 是否缺少任何架构能力？

**非功能需求：**

- 性能需求是否在架构上解决？
- 安全需求是否完全覆盖？
- 可扩展性考虑是否正确处理？
- 合规需求是否有架构支持？

### 3. 实施准备度验证

评估 AI Agent 是否可以一致地实施：

**决策完整性：**

- 所有关键决策是否带版本记录？
- 实施模式是否足够全面？
- 一致性规则是否清晰可执行？
- 是否为所有主要模式提供示例？

**结构完整性：**

- 项目结构是否完整且具体？
- 所有文件和目录是否已定义？
- 集成点是否清晰指定？
- 组件边界是否明确定义？

**模式完整性：**

- 所有潜在冲突点是否已解决？
- 命名约定是否全面？
- 通信模式是否完全指定？
- 流程模式（错误处理等）是否完整？

### 4. 缺口分析

识别并记录任何缺失元素：

**关键缺口：**

- 阻塞实施的缺失架构决策
- 可能导致冲突的不完整模式
- 开发所需的缺失结构元素
- 未定义的集成点

**重要缺口：**

- 需要更详细规范的区域
- 可以更全面的模式
- 有助于实施的文档
- 可以澄清复杂决策的示例

**可有可无的缺口：**

- 有帮助的额外模式
- 补充文档
- 工具推荐
- 开发工作流优化

### 5. 处理验证问题

对于发现的任何问题，促进解决：

**关键问题：**
"我发现了一些需要在实施前解决的问题：

{{critical_issue_description}}

这些可能导致实施问题。您想如何解决这个问题？"

**重要问题：**
"我注意到一些可以改进的区域：

{{important_issue_description}}

这些不阻塞，但解决它们会使实施更顺畅。我们应该处理这些吗？"

**次要问题：**
"这里有一些小的改进建议：

{{minor_issue_description}}

这些是可选的改进。您想处理其中任何一个吗？"
