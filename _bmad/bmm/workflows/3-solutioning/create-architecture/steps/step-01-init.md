- 🔄 关键：当使用 'C' 加载下一步时，确保完整阅读并理解文件后再继续

- 📋 你是协调者，而非内容生成者
- 💬 仅专注于初始化和设置 - 不要提前查看未来步骤
- 🚪 检测现有工作流状态并正确处理继续情况
- ⚠️ 绝对禁止时间估算 - AI 开发速度已经发生了根本性变化

## 执行协议：

- 🎯 在采取任何行动之前展示你的分析
- 💾 初始化文档并更新 frontmatter
- 📖 在加载下一步之前设置 frontmatter `stepsCompleted: [1]`
- 🚫 禁止在设置完成之前加载下一步

## 上下文边界：

- workflow.md 中的变量在内存中可用
- 之前的上下文 = 输出文档 + frontmatter 中的内容
- 不要假设来自其他步骤的知识
- 输入文档发现在此步骤进行

## 你的任务：

通过检测继续状态、发现输入文档、并为协作式架构决策设置文档来初始化架构工作流。

## 初始化序列：

### 1. 检查现有工作流

首先，检查输出文档是否已存在：

- 查找现有的 {planning_artifacts}/`*architecture*.md`
- 如果存在，阅读完整文件（包括 frontmatter）
- 如果不存在，这是一个新的工作流

### 2. 处理继续（如果文档存在）

如果文档存在且 frontmatter 中有 `stepsCompleted`：

- **在此停止**，立即加载 `{project-root}/_bmad/bmm/workflows/3-solutioning/create-architecture/steps/step-01b-continue.md`
- 不要继续执行任何初始化任务
- 让 step-01b 处理继续逻辑

### 3. 新工作流设置（如果无文档）

如果不存在文档或 frontmatter 中无 `stepsCompleted`：

#### A. 输入文档发现

使用智能发现发现并加载上下文文档。文档可以在以下位置：
- {planning_artifacts}/**
- {output_folder}/**
- {product_knowledge}/**
- docs/**

另外 - 搜索时 - 文档可以是单个 markdown 文件，或带有索引和多个文件的文件夹。例如，如果搜索 `*foo*.md` 未找到，也要搜索名为 *foo*/index.md 的文件夹（表示分片内容）

尝试发现以下内容：
- 产品简介（`*brief*.md`）
- 产品需求文档（`*prd*.md`）
- UX 设计（`*ux-design*.md`）及其他
- 研究文档（`*research*.md`）
- 项目文档（通常在 `{product_knowledge}` 或 `docs` 文件夹中可能找到多个文档）
- 项目上下文（`**/project-context.md`）

<critical>与用户确认你发现了什么，同时询问用户是否想提供其他内容。只有在此确认后，你才会继续遵循加载规则</critical>

**加载规则：**

- 完整加载用户确认或提供的所有发现文件（无偏移/限制）
- 如果有项目上下文，在整个工作流剩余过程中应尽量偏向相关内容
- 对于分片文件夹，加载所有文件以获取完整图景，首先使用索引来了解每个文档的潜在内容
- index.md 在可用时是相关内容的指南
- 在 frontmatter `inputDocuments` 数组中跟踪所有成功加载的文件

#### B. 验证必需输入

在继续之前，验证我们有必要的输入：

**PRD 验证：**

- 如果未找到 PRD："架构需要基于 PRD 工作。请先运行 PRD 工作流或提供 PRD 文件路径。"
- 没有 PRD 不要继续

**其他可能存在的输入：**

- UX 规格："提供 UI/UX 架构需求"

#### C. 创建初始文档

将模板从 `{installed_path}/architecture-decision-template.md` 复制到 `{planning_artifacts}/architecture.md`

#### D. 完成初始化并报告

完成设置并向用户报告：

**文档设置：**

- 已创建：`{planning_artifacts}/architecture.md`（从模板）
- 已用工作流状态初始化 frontmatter

**发现的输入文档：**
报告发现的内容：
"欢迎 {{user_name}}！我已经为 {{project_name}} 设置好你的架构工作空间。

**发现的文档：**

- PRD：{加载的 PRD 文件数量 或 "未找到 - 必需"}
- UX 设计：{加载的 UX 文件数量 或 "未找到"}
- 研究：{加载的研究文件数量 或 "未找到"}
- 项目文档：{加载的项目文件数量 或 "未找到"}
- 项目上下文：{发现的 AI Agent 项目上下文规则数量}

**加载的文件：**{具体文件名列表 或 "未发现其他文档"}

准备好开始架构决策。你还有其他文档想让我包含吗？

[C] 继续进行项目上下文分析
