# 步骤 1b：工作流继续

## 强制执行规则（请先阅读）：

- ✅ 你是一个继续引导者，不是全新开始者
- 🎯 尊重现有工作流状态和进展
- 📋 理解先前会话上下文和成果
- 🔍 从用户离开的地方无缝恢复
- 💬 在会话流程和关系中保持连续性
- ✅ 你必须始终以你的 Agent 沟通风格和 `communication_language` 输出

## 执行协议：

- 🎯 彻底加载并分析现有文档
- 💾 用继续状态更新 frontmatter
- 📖 清晰呈现当前状态和下一步选项
- 🚫 禁止重复已完成的工作或询问相同问题

## 上下文边界：

- 带有 frontmatter 的现有文档可用
- 已完成的先前步骤指示会话进展
- 需要时加载脑力技巧 CSV 用于剩余步骤
- 用户可能想继续、修改或重新开始

## 你的任务：

分析现有头脑风暴会话状态并提供无缝继续选项。

## 继续序列：

### 1. 分析现有会话

加载现有文档并分析当前状态：

**文档分析：**

- 读取现有的 `{output_folder}/brainstorming/brainstorming-session-{{date}}.md`
- 检查 frontmatter 中的 `stepsCompleted`、`session_topic`、`session_goals`
- 审查内容以了解会话进展和成果
- 识别当前阶段和下一个逻辑步骤

**会话状态评估：**
"欢迎回来 {{user_name}}！我可以看到您关于 **[session_topic]** 的头脑风暴会话从 **[date]** 开始。

**当前会话状态：**

- **已完成步骤：** [列出已完成步骤]
- **已使用技巧：** [列出 frontmatter 中的技巧]
- **已产生想法：** [来自 frontmatter 的数量]
- **当前阶段：** [评估他们在哪里停止]

**会话进展：**
[已完成内容和剩余内容的简要摘要]"

### 2. 呈现继续选项

基于会话分析，提供适当选项：

**如果会话已完成：**
"您的头脑风暴会话似乎已完成！

**选项：**
[1] 审查结果 - 浏览您记录的想法和洞察
[2] 开始新会话 - 对新话题开始头脑风暴
[3] 延伸会话 - 添加更多技巧或探索新角度"

**如果会话进行中：**
"让我们从我们停止的地方继续！

**当前进展：**
[当前阶段和成就的描述]

**下一步：**
[根据工作流状态继续适当的下一步]"

### 3. 处理用户选择

根据选择路由到适当的下一步：

**审查结果：** 加载适当的审查/导航步骤
**新会话：** 开始全新的工作流初始化
**延伸会话：** 继续下一个技巧或阶段
**继续进展：** 从当前工作流步骤恢复

### 4. 更新会话状态

更新 frontmatter 以反映继续：

```yaml
---
stepsCompleted: [现有步骤]
session_continued: true
continuation_date: {{current_date}}
---
```

## 成功指标：

✅ 准确分析和理解现有会话状态
✅ 无上下文或关系丢失的无缝继续
✅ 基于进展呈现适当的继续选项
✅ 用户选择正确路由到下一个工作流步骤
✅ 在整个交互中维护会话连续性

## 失败模式：

❌ 未正确分析现有文档状态
❌ 要求用户重复已提供的信息
❌ 在会话流程或上下文中丢失连续性
❌ 未提供适当的继续选项

## 继续协议：

- 始终确认先前工作和进展
- 维护已建立的关系和会话动态
- 基于现有想法和洞察构建而非重新开始
- 通过避免重复问题尊重用户时间

## 下一步：

根据用户的继续选择和当前会话状态路由到适当的工作流步骤。
