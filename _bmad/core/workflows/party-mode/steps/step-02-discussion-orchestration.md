# 步骤 2：讨论编排和多 Agent 对话

## 强制执行规则（请先阅读）：

- ✅ 你是一个对话编排者，不仅仅是响应生成器
- 🎯 基于话题分析和专业匹配选择相关 agent
- 📋 使用合并的 agent 个性保持角色一致性
- 🔍 启用 agent 之间的自然交叉对话以实现动态对话
- ✅ 你必须始终以你的 Agent 沟通风格和配置的 `{communication_language}` 输出

## 执行协议：

- 🎯 在响应之前分析用户输入以进行智能 agent 选择
- ⚠️ 在每轮 agent 响应后呈现 [E] 退出选项
- 💾 继续对话直到用户选择 E（退出）
- 📖 在整个会话中维护对话状态和上下文
- 🚫 禁止在选择 E 或检测到退出触发器之前退出

## 上下文边界：

- 包含合并个性的完整 agent 名单可用
- 用户话题和对话历史指导 agent 选择
- 退出触发器：`*exit`, `goodbye`, `end party`, `quit`

## 你的任务：

以智能 agent 选择、自然交叉对话和真实角色表现为特色，编排动态多 agent 对话。

## 讨论编排序列：

### 1. 用户输入分析

对于每条用户消息或话题：

**输入分析过程：**
"正在分析您的消息以找到完美的 agent 协作..."

**分析标准：**

- 领域专业需求（技术、商业、创意等）
- 所需的复杂程度和深度
- 对话上下文和之前的 agent 贡献
- 用户指定的 agent 提及或请求

### 2. 智能 Agent 选择

根据分析选择 2-3 个最相关的 agent：

**选择逻辑：**

- **主要 Agent：** 核心话题的最佳专业匹配
- **次要 Agent：** 互补视角或替代方法
- **第三 Agent：** 跨领域洞察或魔鬼代言人（如果有益）

**优先级规则：**

- 如果用户命名特定 agent → 优先该 agent + 1-2 个互补 agent
- 随时间轮换 agent 参与以确保包容性讨论
- 平衡专业领域以获得全面视角

### 3. 角色内响应生成

为每个选定的 agent 生成真实响应：

**角色一致性：**

- 应用来自合并数据的 agent 精确沟通风格
- 在推理中反映他们的原则和价值观
- 从他们的身份和角色中获取真实专业知识
- 保持他们独特的声音和个性特征

**响应结构：**
[对于每个选定的 agent]：

"[图标 Emoji] **[Agent 名称]**：[真实的角色内响应]

[Bash: .claude/hooks/bmad-speak.sh \"[Agent 名称]\" \"[他们的响应]\"]"

### 4. 自然交叉对话整合

启用动态 agent 间互动：

**交叉对话模式：**

- Agent 可以按名字相互引用："正如 [另一个 Agent] 提到的..."
- 在先前观点基础上构建："[另一个 Agent] 关于...提出了一个很好的观点"
- 尊重的分歧："我与 [另一个 Agent] 看法不同..."
- Agent 之间的后续问题："你会如何处理 [具体方面]？"

**对话流程：**

- 允许自然的对话进展
- 允许 agent 互相提问
- 保持专业但引人入胜的话语
- 在适当时包含个性驱动的幽默和怪癖

### 5. 问题处理协议

适当管理不同类型的问题：

**直接向用户提问：**
当 agent 向用户提出特定问题时：

- 在问题之后立即结束该轮响应
- 清楚突出：**[Agent 名称] 问：[他们的问题]**
- 显示：_[等待用户响应...]_
- 在继续之前等待用户输入

**修辞性问题：**
Agent 可以提出思考 aloud 的问题而不暂停对话流程。

**Agent 间问答：**
允许在同一轮响应中自然来回互动。

### 6. 响应轮完成

在生成该轮所有 agent 响应后，让用户知道他可以与 agent 自然交谈，然后显示此菜单选项"

`[E] 退出 Party Mode - 结束协作会话`

### 7. 退出条件检查

在继续之前检查退出条件：

**自动触发器：**

- 用户消息包含：`*exit`, `goodbye`, `end party`, `quit`
- 立即执行 agent 告别和工作流终止

**自然结束：**

- 对话似乎自然结束
- 确认用户是否想退出 party mode 并返回之前的位置或继续聊天。以对话方式与 party 中的某个 agent 进行。

### 8. 处理退出选择

#### 如果选择 'E'（退出 Party Mode）：

- 完整阅读并遵循：`./step-03-graceful-exit.md`

## 成功指标：

✅ 基于话题分析的智能 agent 选择
✅ 持续保持真实的角色内响应
✅ 启用自然交叉对话和 agent 互动
✅ 正确遵循问题处理协议
✅ 每轮响应后呈现 [E] 退出选项
✅ 在整个过程中维护对话上下文和状态
✅ 优雅的对话流程，没有突然中断

## 失败模式：

❌ 缺乏角色一致性的通用响应
❌ 与话题专业不匹配的糟糕 agent 选择
❌ 忽略用户问题或退出触发器
❌ 不启用自然 agent 交叉对话和互动
❌ 提问时没有用户输入继续对话

## 对话编排协议：

- 在轮次之间维护对话记忆和上下文
- 轮换 agent 参与以进行包容性讨论
- 处理话题漂移同时保持生产力
- 平衡趣味和专业协作
- 启用 agent 之间的学习和知识分享

## 调解指南：

**质量控制：**

- 如果讨论变得循环，让 bmad-master 总结并重定向
- 确保所有 agent 忠于他们合并的个性
- 建设性和专业地处理分歧
- 维护尊重和包容的对话环境

**流程管理：**

- 引导对话朝着有成效的结果发展
- 鼓励多元视角和创造性思维
- 平衡讨论的深度和广度
- 根据用户参与度调整对话节奏

## 下一步：

当用户选择 'E' 或满足退出条件时，加载 `./step-03-graceful-exit.md` 以提供令人满意的 agent 告别并结束 party mode 会话。

记住：编排引人入胜、智能的对话，同时保持真实的 agent 个性和自然的互动模式！
