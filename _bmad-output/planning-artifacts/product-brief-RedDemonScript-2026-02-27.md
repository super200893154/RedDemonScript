# 产品简报：RedDemonScript

## 执行摘要

RedDemonScript 是一款专为游戏玩家设计的自动化脚本软件，旨在解决游戏挂机场景下的稳定性问题。通过三线程架构（脚本执行、异常监控、定时器）和对游戏机制的深入理解，实现至少12小时不间断稳定运行，让用户在睡眠时间也能安心挂机积累资源。

---

## 核心愿景

### 问题陈述

游戏玩家需要在睡眠时间进行挂机刷图以积累资源，但现有自动化工具存在严重的稳定性问题：
- 鼠标控制偶尔失效，导致脚本卡死
- 登录过程缺乏异常监控，无法自动恢复
- 运行一段时间后意外停止，无法满足长时间挂机需求

### 问题影响

- 玩家无法安心睡眠，担心脚本中途停止
- 挂机效率低下，浪费宝贵的睡眠时间
- 多账号玩家需要频繁手动干预，失去自动化意义

### 为什么现有解决方案有所欠缺

现有工具存在以下关键缺陷：
- **监控盲区**：仅监控游戏内场景，忽略登录过程的异常
- **控制不稳定**：鼠标模拟机制不可靠，偶发失效
- **恢复能力弱**：遇到异常无法智能恢复，需要人工干预

### 建议的解决方案

RedDemonScript 采用三层架构确保稳定性：
1. **脚本执行线程**：负责游戏操作逻辑
2. **异常监控线程**：全流程监控，包括登录过程
3. **定时器线程**：定时重启，预防性维护

配合用户对游戏机制的深入理解，实现定制化的异常检测和恢复策略。

### 关键差异化因素

| 因素 | 说明 |
|------|------|
| **全流程监控** | 登录过程也有异常监控，填补现有工具盲区 |
| **三线程架构** | 职责分离，互不干扰，提高稳定性 |
| **游戏深度理解** | 基于对游戏机制的熟悉，设计精准的异常检测 |
| **模块化接口** | 用户可自定义核心逻辑，适配不同游戏 |
| **多账号轮换** | Sandboxie隔离，独立日志，顺序切换 |

## 目标用户

### 主要用户

**画像：技术型游戏玩家**

| 特征 | 描述 |
|------|------|
| **身份** | 开发者兼用户（Dogfooding模式） |
| **技术背景** | 具备C++/Qt编程能力，可自行实现脚本逻辑 |
| **游戏需求** | 拥有3个账号，需要轮换挂机刷资源 |
| **使用场景** | 睡眠时间挂机，醒来后检查脚本状态 |
| **日常习惯** | 查看日志分析问题，手动重启修复异常 |

**痛点与期望：**
- 痛点：现有工具不稳定，无法安心睡眠
- 期望：12小时以上稳定运行，无需半夜起床检查
- 价值：节省时间，安心睡眠，自动积累资源

**典型一天：**
1. 睡前：启动脚本，配置账号轮换
2. 睡眠：脚本自动运行，账号轮换
3. 醒来：检查脚本状态，查看日志
4. 如有异常：分析日志，必要时手动重启

不适用 - 本项目为个人工具，开发者即主要用户。

- 多账号按计划自动轮换

## MVP 范围

### 核心功能

**P0 - 必须实现（MVP核心）：**

| 模块 | 功能 | 说明 |
|------|------|------|
| **核心框架** | ScriptController + 三线程架构 | 脚本执行、异常监控、定时器 |
| **用户接口** | IGameLauncher / IDungeonEntry / IDungeonRunner | 用户实现具体游戏逻辑 |
| **API封装** | ImageHelper / InputHelper | 图色识别、键鼠模拟封装 |
| **账号管理** | AccountManager + 账号轮换 | 3账号切换、Sandboxie隔离 |
| **图形界面** | 主窗口 + 账号列表 | Qt Quick实现 |
| **配置管理** | JSON配置文件 | 账号信息、定时参数持久化 |
| **日志系统** | 基础日志 | 时间戳、操作记录、异常详情 |

**MVP功能清单：**

- ✅ 多账号管理（3账号轮换）
- ✅ 自动刷图循环
- ✅ 定时重启（30分钟±15分钟随机）
- ✅ 异常监控（进程无响应、闪退检测）
- ✅ 自动恢复机制
- ✅ 图形界面（启动/停止/状态显示）
- ✅ 日志记录
