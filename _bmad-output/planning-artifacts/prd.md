## 执行摘要

RedDemonScript 是一款 Windows 桌面自动化脚本软件，专为游戏挂机场景设计。核心目标是实现 **≥12小时不间断稳定运行**，让用户在睡眠时间也能安心挂机积累资源。

### 问题与解决方案

现有自动化工具存在稳定性问题：鼠标控制偶发失效、登录过程缺乏异常监控、运行一段时间后意外停止。用户无法安心睡眠，挂机效率低下。

RedDemonScript 采用**三线程架构**解决这些问题：
- **脚本执行线程**：负责游戏操作逻辑
- **异常监控线程**：全流程监控，包括登录过程
- **定时器线程**：定时重启，预防性维护（30分钟±15分钟随机）

配合用户可自定义的接口设计，框架提供稳定性保障，用户填充具体游戏逻辑。

### 特别之处

| 差异化因素 | 说明 |
|------------|------|
| **全流程监控** | 登录过程也有异常监控，填补现有工具盲区 |
| **三线程架构** | 职责分离，互不干扰，提高稳定性 |
| **用户可定制** | 框架提供核心稳定性，用户实现游戏逻辑接口 |
| **多账号轮换** | Sandboxie隔离，独立日志，顺序切换 |

### 目标用户

**开发者兼用户（Dogfooding模式）**：具备C++/Qt编程能力的技术型游戏玩家，拥有多个游戏账号需要轮换挂机。

### 成功愿景

醒来发现脚本稳定运行整晚，日志清晰记录所有操作和异常，多账号按计划自动轮换。

## 用户旅程

### 用户画像

**技术型游戏玩家（ckm）**
| 特征 | 描述 |
|------|------|
| **身份** | 开发者兼用户（Dogfooding模式） |
| **技术背景** | 具备C++/Qt编程能力 |
| **游戏需求** | 拥有3个账号，需要轮换挂机刷资源 |
| **使用场景** | 睡眠时间挂机，醒来后检查脚本状态 |
| **核心痛点** | 现有工具不稳定，无法安心睡眠 |

### 旅程 1：成功路径 - 安心挂机

**开场场景：** 晚上11点，用户准备睡觉。打开 RedDemonScript，配置好3个账号的轮换顺序，点击"开始"。

**上升行动：**
- 脚本自动启动第一个游戏账号（通过 Sandboxie 隔离）
- 脚本执行线程开始刷图循环
- 异常监控线程持续检测游戏状态
- 定时器线程在28分钟后触发重启（随机间隔）

**高潮：** 凌晨3点，游戏意外闪退。监控线程立即检测到异常，自动重启游戏并恢复运行。用户睡得很安稳，毫不知情。

**结局：** 早上7点，用户醒来。打开电脑，看到脚本仍在稳定运行。日志清晰记录了整晚的操作：完成副本 47 次，自动恢复异常 1 次。3个账号按计划轮换完毕。

### 旅程 2：边缘情况 - 异常排查

**开场场景：** 早上醒来，用户发现脚本已停止，最后一个账号没有完成轮换。

**上升行动：**
- 查看日志，发现"进程无响应"警告
- 日志显示最后一次操作是切换账号时卡住
- 问题定位时间：3分钟（时间戳 + 操作记录）

**高潮：** 用户意识到是 Sandboxie 沙箱配置问题，修复后手动重启脚本。

**结局：** 问题快速定位并解决，恢复正常挂机。

### 旅程 3：配置与维护

**开场场景：** 周末，用户想添加一个新账号到轮换列表。

**上升行动：**
- 打开配置界面，添加新账号信息
- 设置最大运行时间和轮换顺序
- 配置持久化保存

**结局：** 新账号成功加入轮换，下次启动自动生效。

| 旅程 | 揭示的能力需求 |

| 成功路径 | 自动刷图循环、异常监控、定时重启、账号轮换、日志记录 |

## 功能需求

### 核心脚本执行

- FR1: 用户可以通过实现 IGameLauncher 接口定义游戏启动逻辑
- FR2: 用户可以通过实现 IDungeonEntry 接口定义副本入口逻辑
- FR3: 用户可以通过实现 IDungeonRunner 接口定义副本战斗逻辑
- FR4: 系统可以循环执行刷图流程直到收到停止指令
- FR5: 系统可以调用外部 DLL 进行图色识别操作
- FR6: 系统可以调用外部 DLL 进行键鼠模拟操作

### 异常监控与恢复

- FR7: 系统可以检测游戏进程是否无响应
- FR8: 系统可以检测游戏进程是否意外终止（闪退）
- FR9: 系统可以在检测到异常后自动重启游戏并恢复运行
- FR10: 系统可以在异常恢复时记录异常详情到日志

### 定时器管理

- FR11: 系统可以在配置的时间间隔后触发定时重启
- FR12: 系统可以在定时重启时间上添加随机偏移（模拟真人行为）

### 多账号管理

- FR13: 用户可以添加、编辑、删除账号信息
- FR14: 用户可以配置账号的最大运行时间
- FR15: 用户可以配置账号的轮换顺序
- FR16: 用户可以启用或禁用特定账号
- FR17: 系统可以按配置顺序自动轮换账号
- FR18: 系统可以在 Sandboxie 沙箱中启动游戏实例
- FR19: 系统可以在账号切换时终止当前沙箱中的所有进程
- FR20: 系统可以为每个账号维护独立的日志存储

### 图形界面

- FR21: 用户可以通过主窗口启动脚本执行
- FR22: 用户可以通过主窗口停止脚本执行
- FR23: 用户可以查看当前运行状态（运行中/已停止/异常）
- FR24: 用户可以查看账号列表及其状态
- FR25: 用户可以查看当前运行的账号信息

### 日志系统

- FR26: 系统可以记录带时间戳的操作日志
- FR27: 系统可以记录异常事件详情
- FR28: 系统可以按账号隔离存储日志
- FR29: 用户可以查看当前账号的运行日志
- FR30: 系统可以自动清理超过 7 天的日志文件

### 配置管理

- FR31: 系统可以从 JSON 文件加载配置
- FR32: 系统可以将配置保存到 JSON 文件
- FR33: 用户可以配置定时重启的时间间隔
- FR34: 用户可以配置 Sandboxie 安装路径
- FR35: 用户可以配置游戏进程名称
